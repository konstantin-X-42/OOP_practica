## **2. модуль ```classes.py``` в пакете ```src```**
### ***Классы и объекты ООП***
    Классы не просто хранят данные, но и динамически отслеживают состояние всей системы
    (ведут сквозной подсчет созданных объектов) за счет механизмов инкапсуляции и атрибутов класса.

    1. Класс Product (Модель товара). Отвечает за создание и хранение информации о конкретной торговой единице.
        - Тип данных: Объектный класс (динамическая модель).
        - Свойства объекта (Атрибуты экземпляра):
            - name (строка) — уникальное коммерческое наименование товара.
            - description (строка) — подробное описание характеристик товара.
            - price (вещественное число) — текущая стоимость единицы товара.
            - quantity (целое число) — доступный остаток товара на складе.

    2. Класс Category (Модель категории). Выступает в роли контейнера-группировщика для объектов класса Product. Помимо хранения локальных данных, класс выполняет аналитическую функцию — отслеживает глобальное состояние всей торговой системы.
        - Свойства объекта (Атрибуты экземпляра):
            - name (строка) — название товарной группы (например, «Смартфоны»).
            - description (строка) — описание группы товаров.
            - products (список объектов) — коллекция экземпляров класса Product, входящих в эту категорию.
        - Глобальные свойства (Атрибуты класса / Статические переменные):
            - category_count (целое число) — счетчик, фиксирующий общее количество созданных категорий в системе.
            - product_count (целое число) — счетчик, фиксирующий суммарное количество уникальных товаров, распределенных по всем категориям.

    Логика работы сквозных счетчиков (Конструктор Category). При инициализации каждого нового объекта категории (__init__) автоматически срабатывает механизм агрегации данных, основанный на изменении состояния самого класса:
        - Инкремент категорий (Category.category_count += 1):
            Каждый раз, когда создается новая категория, этот счетчик увеличивается на 1. Переменная связана напрямую с именем класса Category, что позволяет сохранять значение между созданиями разных объектов.
        - Агрегация количества товаров (Category.product_count += len(products)):
            Конструктор принимает на вход список товаров products. Встроенная функция len() вычисляет количество элементов (товаров) в этом списке. Оператор += прибавляет полученное число к общему глобальному счетчику. Таким образом, система в реальном времени знает, сколько всего товаров загружено в магазин.

## Тестирование

В проекте используется фреймворк `pytest` для проверки корректности работы функций и обработки ошибок.

### Запуск тестов
Команда выводит информацию в консоль, автоматически запускает все тесты в модулях (во всем проекте) с помощью фреймворка pytest внутри изолированного виртуального окружения Poetry.
```
poetry run pytest
```
Команда выводит информацию в консоль, автоматически запускает тесты в указанном модуле ```tests``` - папка с модулем теста, ```test_decorators.py``` - название модуля тестов
```
pytest tests/test_decorators.py
```
Команда выводит описание ошибки и информацию в консоль, автоматически запускает тесты в указанном модуле
```
pytest -s tests/test_decorators.py
```

### Анализ покрытия кода (Coverage)
Посмотреть, насколько полно код покрыт тестами, используйте `pytest-cov`:
```
poetry run pytest --cov=src
```
Команда автоматически запускает все тесты в проекте (из папки tests) и выводит в консоль сводную таблицу с процентом покрытия кода для модулей внутри директории ```src```
```
pytest --cov=src tests/
```
Команда автоматически запускает все тесты в проекте (из папки tests) и генерирует детальный интерактивный HTML-отчет с процентом покрытия кода для модулей внутри директории ```src```
```
pytest --cov=src --cov-report=html tests/
```
