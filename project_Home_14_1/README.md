## Установка библиотек:
1. Библиотека фреймворк ```pytest``` — зависимость для тестирования, проверки стиля кода или отладки, инструмент для написания автоматических тестов. Каманда, установка через инструмент для управления проектами и зависимостями — ```poetry```

    ```
    poetry add --group dev pytest
    ```
3. Библиотека ```pytest-cov``` — зависимость для тестирования, проводит анализ покрытия кода при проведении тестирования. Каманда, установка через ```poetry```
    ```
    poetry add --group dev pytest-cov
    ```
4. Библиотека ```requests``` — инструмент в Python для отправки HTTP-запросов в интернет. Каманда, установка через ```poetry```
    ```
    poetry add requests
    ```
5. Библиотека ```python-dotenv``` — безопасно разделяет написаный программный код и конфиденциальные данные (пароли, токены, ключи API), автоматически находит текстовый файл ```.env```.  Каманда, установка через ```poetry```
    ```
    poetry add python-dotenv
    ```
6. Библиотека ```pandas```  — основной пакет анализа данных в Python, считывает таблицы из файлов во внутренний объект DataFrame (двумерную таблицу в оперативной памяти, аналогичную листу Excel)
    ```
   poetry add pandas
    ```
7. библиотека ```openpyxl``` — дополнительный пакет (движок) для pandas, необходимый для работы со структурой файлов формата .xlsx и .xlsm. Он позволяет считывать конкретные листы (вкладки) книги Excel в объект DataFrame, а также сохранять (записывать) данные из DataFrame обратно в файлы Excel
    ```
   poetry add openpyxl
    ```
8.  библиотека ```pandas-stubs``` — официальный пакет, устанавливается исключительно в окружение для разработки (dev-зависимости), который содержит аннотации типов (type hints) для библиотеки pandas
    ```
    poetry add --group dev pandas-stubs
    ```

### ***1. маскировка цифр номера карты пользователя в формате: 8990 92** **** 5229***
```

```

- строку - ```str```
- набор целых чисел - ```int```

- **Error_01** - ошибочное количество символов в номере карты
- **Error_02** - введён не допустимый тип данных или не числовой символ в номере карты
- **Error_03** - не введён номер карты
###### Аргументы функции:
```
    get_mask_card_number('7000792289606361')
```
###### функция возвращает строку (str) скрытого номера карты:
```
 7000 79** **** 6361
```
##### В функцию импортирован модуль log_config.py, ведёт логирование этапов исполнения функции в директорию: logs/masks.log

## Тестирование

В проекте используется фреймворк `pytest` для проверки корректности работы функций и обработки ошибок.

### Запуск тестов
Команда выводит информацию в консоль, автоматически запускает все тесты в модулях (во всем проекте) с помощью фреймворка pytest внутри изолированного виртуального окружения Poetry.
```
poetry run pytest
```
Команда выводит информацию в консоль, автоматически запускает тесты в указанном модуле ```tests``` - папка с модулем теста, ```test_decorators.py``` - название модуля тестов
```
pytest tests/test_decorators.py
```
Команда выводит описание ошибки и информацию в консоль, автоматически запускает тесты в указанном модуле
```
pytest -s tests/test_decorators.py
```

### Анализ покрытия кода (Coverage)
Посмотреть, насколько полно код покрыт тестами, используйте `pytest-cov`:
```
poetry run pytest --cov=src
```
Команда автоматически запускает все тесты в проекте (из папки tests) и выводит в консоль сводную таблицу с процентом покрытия кода для модулей внутри директории ```src```
```
pytest --cov=src tests/
```
Команда автоматически запускает все тесты в проекте (из папки tests) и генерирует детальный интерактивный HTML-отчет с процентом покрытия кода для модулей внутри директории ```src```
```
pytest --cov=src --cov-report=html tests/
```
